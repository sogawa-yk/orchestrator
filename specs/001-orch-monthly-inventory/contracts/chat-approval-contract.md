# チャット / 承認 UI 契約

## 1. 対象

ORCH が利用者に提示するチャット応答、深掘り表示、通知承認 UI の最小契約を定義する。

## 2. 初回要約応答

### 必須項目

- 全体サマリ
- 上位 3〜5 件の要対応項目
- 各項目の優先理由
- 次アクション候補
- 深掘り可能な案内

### 表示ルール

- 初回は問題単位の一覧を既定とする
- 1 項目ごとに「何が問題か」「なぜ優先か」「次に何ができるか」を含める
- 利用者が次に入力すべき例を 1 つ以上含める

## 3. 深掘り応答

### 入力例

- この件を詳しく見せて
- コスト起点で並べ替えて
- このコンパートメントだけ見せて
- 誰に連絡が必要か見せて

### 契約

- 既存のセッション前提を引き継ぐこと
- 表示ビューの切り替え結果を明示すること
- 利用者が次に取れる深掘り行動を残すこと

## 4. 承認 UI

### 表示必須項目

- 宛先メールアドレス
- 件名
- 本文
- 対象 issue
- 送信理由
- 選択可能な操作

### 選択可能な操作

- 承認
- 却下
- 修正指示
- 保留

### 契約

- 承認されるまでは外部送信を実行してはならない
- 保留時は再開に必要な情報を保持する
- 失敗時は失敗理由と再試行可否を表示する
