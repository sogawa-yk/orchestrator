# ORCH プロジェクト憲章

## 中核原則

### I. 日本語ドキュメント原則
仕様書、計画書、タスクリスト、チェックリスト、設計判断、レビュー記録、運用手順書は
すべて日本語で作成しなければならない。英語の固有名詞、API 名、ライブラリ名、
Kubernetes や Playwright のような一般化した技術用語は併記可能だが、判断根拠と
期待動作の説明は日本語で残さなければならない。日本語化されていない文書は
レビュー完了扱いにしてはならない。

### II. 仕様駆動トレーサビリティ原則
すべての機能開発は `spec.md`、`plan.md`、`tasks.md` を起点に進め、要件、実装、
検証結果の対応関係を追跡可能にしなければならない。各ユーザーストーリーは独立して
価値を提供し、受け入れ条件、ローカルテスト、Kubernetes 上の検証シナリオへ
明示的にひも付いていなければならない。追跡不能な要求、根拠のない実装、検証計画の
ない変更は許可しない。

### III. ローカル単体・結合テスト必須原則
実装対象のすべての変更は、ローカル環境で単体テストと結合テストを必ず実施し、
成功結果を確認してから次工程へ進まなければならない。テストの省略、未実行、
失敗の黙認、後回しの是認は禁止する。新規機能、既存改修、バグ修正のいずれでも、
対象範囲に応じた単体テストと結合テストを追加または更新し、回帰防止を担保する。

### IV. Kubernetes 配備後 E2E 検証原則
ローカルテスト完了後、変更は Kubernetes クラスタ上の検証環境へ配備し、
Playwright によるブラウザ操作ベースの E2E テストを実施しなければならない。
E2E は想定利用シナリオを 20 件以上含み、正常系、異常系、権限制御、承認操作、
リトライ、主要導線の回帰を網羅しなければならない。20 シナリオ未満、または
未解決の失敗を残した状態では、受け入れ、デモ、リリースに進んではならない。

### V. 運用安全性と観測性原則
マルチエージェント連携、HTTP ベース Task API、Chainlit UI、Oracle Database、
Kubernetes 運用を前提とする実装は、失敗原因を追跡できるログ、メトリクス、
トレースまたは同等の観測情報を備えなければならない。デプロイ後に E2E 失敗時の
切り分けができない変更、あるいはロールバック判断に必要な情報を欠く変更は
完成と見なさない。観測性は品質保証の一部であり、任意作業ではない。

## 追加制約

- 生成される仕様系ドキュメントは日本語を標準とし、テンプレートも日本語で維持する。
- 検証計画には最低でもローカル単体テスト、ローカル結合テスト、Kubernetes 検証環境
  への配備手順、Playwright E2E 20 シナリオ以上の一覧を含める。
- E2E シナリオは機能の主要価値、承認フロー、エラー表示、再実行、外部エージェント
  連携の各観点をカバーする。
- テスト結果はレビュー時に参照できる形で記録し、失敗時の暫定回避ではなく根本原因の
  修正を優先する。

## 開発・検証ワークフロー

1. 要求整理時に `spec.md` を日本語で作成し、受け入れ条件と検証観点を定義する。
2. `plan.md` で技術方針、対象環境、テスト戦略、Kubernetes 配備方法を確定する。
3. `tasks.md` では単体、結合、Kubernetes 配備、Playwright E2E を個別タスクとして
   明示し、依存関係を定義する。
4. 実装後はローカルで単体テストと結合テストを完了し、失敗がないことを確認する。
5. その後 Kubernetes クラスタへ配備し、Playwright で 20 シナリオ以上の E2E を実行する。
6. レビューでは文書の日本語整備、要件トレーサビリティ、各テスト実行結果、観測性の
   妥当性を確認し、未達があれば差し戻す。

## ガバナンス

この憲章は本リポジトリにおける開発、レビュー、計画、検証の最上位規範とする。
例外を認める場合は、変更理由、影響範囲、代替策、期限付き是正計画を同一変更内で
文書化し、承認者が明示的に承認しなければならない。憲章改定時は関連テンプレート、
ガイダンス文書、運用文書を同じ変更セットで同期更新すること。

バージョン管理はセマンティックバージョニングに従う。原則の削除や後方非互換な
統治変更は MAJOR、原則や必須手順の追加・実質拡張は MINOR、表現修正や曖昧さ解消は
PATCH を増分する。すべてのレビュー、計画、実装完了判定では、この憲章への適合確認を
必須項目とし、不適合は修正または正式な例外承認が完了するまで未完了扱いとする。
